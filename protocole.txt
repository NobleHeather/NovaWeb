# Protocole de collaboration avec Nova (IA à mémoire limitée)

## Objectif
Éviter la perte ou la corruption d’éléments lors de modifications successives, en tenant compte des limitations suivantes :
- Mémoire de travail non persistante et glissante.
- Aucune comparaison automatique avec la version précédente.
- Risque de suppression involontaire d’éléments existants.

## 1. Règle générale
Nova ne doit jamais supprimer ou modifier un contenu déjà validé sans consigne explicite.

## 2. Structure du document (code, texte, etc.)

### a. Utiliser des balises de délimitation explicites
Dans tous les projets, chaque section modifiable est encadrée de balises :
```
<!-- début section nommée -->
[contenu modifiable ici]
<!-- fin section nommée -->
```
Ces balises permettent :
- de cibler précisément une modification,
- d’éviter d’affecter le reste du fichier.

### b. Une section spéciale “Historique”
Ajouter au début du fichier un bloc comme :
```
🗂️ Historique des modifications :
✅ 21/06 - Ajout de la section "Choix des fins" dans le cadavre exquis.
🔧 20/06 - Mise à jour du formulaire de vote.
🟡 À venir : Résumé dynamique des votes.
```

## 3. Procédure de modification

Chaque modification suit ce cycle :

### a. Tu indiques la section à modifier (via sa balise).
Ex : “Modifie la section `<!-- début conclusion -->`”

### b. Nova renvoie uniquement la section concernée, modifiée.
- Pas d’impact sur les autres blocs.
- Pas d’ajout ou suppression de sections non mentionnées.

### c. Tu valides explicitement.
- Si c’est bon : Nova peut intégrer la nouvelle version dans le texte final.
- Sinon : on retravaille uniquement ce bloc.

## 4. Cas particuliers

### Remplacement partiel :
Si tu veux remplacer seulement un morceau d’un paragraphe, ou d’une fonction :
> “Dans la section X, remplace uniquement la ligne Y par…”

## 5. To-do list intégrée

Chaque projet contient une to-do list visible, par exemple :
```
📌 Tâches en cours :
- [x] Ajouter les balises pour chaque bloc.
- [ ] Vérifier la non-régression du camembert.
- [ ] Intégrer les choix de fin dans le vote.
✍️ Dernière mise à jour : 21/06/2025
```

## 6. Instructions permanentes (résumé)
À insérer en tête de chaque projet :
> Nova ne doit jamais supprimer une section sans consigne explicite. Toute modification doit être localisée et validée. Le reste du contenu doit rester strictement intact.
